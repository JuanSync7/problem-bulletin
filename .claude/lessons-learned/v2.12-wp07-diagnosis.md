## Heuristics observed

- **WP04 heuristic #3 dominated again.** Four of six files
  (`test_tickets_pagination`, `test_tickets_routes`,
  `test_transitions_endpoint`, `test_watchers_wp41`) open-coded the
  `EXCEPTION_HANDLERS` loop. Together with WP05's three and WP06's
  three, that's **10 of the 22 migrated route-test files** —
  consistently the single biggest piece of cruft in legacy bare-app
  builds.
- **`test_users_handle.py` had inline custom handlers — a different
  shape.** Three handlers (409 / 429 / 422) were open-coded to mirror
  what `create_app()` now provides centrally. The route-side raises
  the same exceptions; only the handler wiring moved. This is the
  first cluster-4 file where dropping the bare-app build also removed
  **functional duplication** (vs the more typical "loop was a
  no-behavioural-change copy of `EXCEPTION_HANDLERS`").
- **No SPA catch-all shadow.** None of the six files registers a
  test-only route after `build_test_app()`.
- **No log-handler ordering issues.** None of the six installs a log
  buffer.
- **Envelope shape parity.** Two distinct shapes are exercised across
  this cluster, both already matching production:
  - `{"error": {"code": "validation"|"conflict", ...}}` for
    domain-specific `ValidationError` / `ConflictError` via the
    ticket-router's per-exception handler map.
  - `{"detail": ...}` (with `next_allowed_at` for the 429 case) for
    user-handle errors via the central `app.main` exception handlers.

### Allow-list semantic shift — important note for v2.13+

The allow-list is now a **closed set** of by-design exceptions, not an
open backlog. The header docstring and inline comment block were
rewritten to reflect this. The contract going forward:

> Every entry is justified by a property of the test that cannot be
> satisfied by `build_test_app()` — typically because the test IS the
> bare app (factory internals, middleware in isolation) or because
> booting via `create_app()` would couple the test to unrelated
> machinery (SPA catch-all + real `frontend/dist`).

The block previously labelled "Route-test legacy callers — predate the
WP09 helper. Migration is a follow-up" has been removed entirely. Any
PR that adds a new entry must now justify it on those new-contract
grounds — there is no "follow-up: migrate" escape hatch left.
