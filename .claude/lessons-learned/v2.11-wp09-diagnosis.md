## C3 — Route-test factories must use `app.main.create_app()`

### Approach taken
Shipped **both** (a) the lint AND (b) the helper, but did NOT migrate the legacy callers — they are documented in the lint's explicit allow-list as pre-WP09 legacy with a "migrate to `build_test_app()`" follow-up. The migration scope (36 files) was too large for this WP; the lint freezes the surface and the helper unblocks future migration.

### Artifacts
- `tests/helpers/app_factory.py::build_test_app(**overrides)` — delegates to `create_app()`, merges `dependency_overrides` kwarg.
- `tests/test_create_app_factory_lint_wp09.py` — AST scanner over `tests/**/*.py` flagging `Call(func=Name('FastAPI'))`. Allow-list of 37 files (36 legacy + the helper file is auto-skipped).
- 4 tests:
  - `test_no_bare_fastapi_constructor_outside_allowlist` — the main lint
  - `test_create_app_factory_lint_detects_synthetic_bad` — self-test (3 cases)
  - `test_build_test_app_helper_wires_exception_handlers` — load-bearing smoke: asserts `AppError` handler + every `_TICKET_EXC_HANDLERS` key registered
  - `test_build_test_app_supports_dependency_overrides` — kwarg contract

### Allow-list size
37 files, grouped:
- **Self-tests** (2): `test_main.py`, `test_v2_11_wp05_boot_hardening.py` — must not recurse via `create_app()`.
- **Middleware isolation** (4): `middleware/test_bearer_auth.py`, `middleware/test_correlation.py`, `middleware/test_security.py`, `observability/test_otel_init.py` — minimal apps deliberately exclude unrelated middleware.
- **Legacy route-tests** (31): every `tests/routes/test_*.py` and `tests/e2e/test_*.py` site that predates the helper. Each entry justifies "legacy (pre-WP09); follow-up: migrate to build_test_app()".

### Sites found / migrated / allow-listed
- Found: 36 bare-FastAPI files (37 once `test_agents_activity.py` was added after first lint run flagged it).
- Migrated: 0 (deliberate — scope too large; helper proves the path).
- Allow-listed: 37.

### Bugs surfaced
None during this WP — the lint is a freeze, not a migration. The follow-up migration may surface latent silent-pass bugs in the allow-listed tests (the whole reason for the lint).

## C6 — Conftest ambient-env audit

### Scope
Only `tests/conftest.py` carries `os.environ.setdefault(...)` calls. The 5 sub-conftests (`middleware/`, `mcp_server/`, `services/`, `observability/`, `routes/`) have none.

### Classification table

| Key | Value | Settings model default | Classification | Annotation added |
|-----|-------|------------------------|----------------|------------------|
| `DATABASE_URL` | `postgresql+asyncpg://test:test@localhost:5432/test_aion` | none (required) | load-bearing: required to construct engine | Y |
| `AZURE_TENANT_ID` | `test-tenant-id` | none (required) | load-bearing: required by Settings | Y |
| `AZURE_CLIENT_ID` | `test-client-id` | none (required) | load-bearing: required by Settings | Y |
| `AZURE_CLIENT_SECRET` | `test-client-secret` | none (required) | load-bearing: required by Settings | Y |
| `JWT_SECRET` | `test-jwt-secret-at-least-32-chars-long` | none (required, min-length) | load-bearing: required, min-length enforced | Y |
| `SMTP_HOST` | `localhost` | none (required) | load-bearing: required for notification config | Y |
| `SMTP_PORT` | `587` | none (required) | load-bearing: required for notification config | Y |
| `SMTP_FROM` | `test@aion-bulletin.local` | none (required) | load-bearing: required for notification config | Y |
| `BASE_URL` | `http://localhost:8000` | none (required) | load-bearing: used in magic-link URLs | Y |
| `STORAGE_PATH` | `/tmp/aion-test-storage` | none (required) | load-bearing: attachment storage root | Y |
| `ENVIRONMENT` | `development` | `development` | load-bearing: matches model default; pinned to keep `DEV_AUTH_BYPASS=false` legal (WP05 fail-fast would refuse to boot otherwise) | Y |
| `DEV_AUTH_BYPASS` | `false` | `False` | load-bearing: matches model default; explicit to document intent | Y |
| `APP_NAME` | `Aion Bulletin Test` | `"Aion Bulletin"` | load-bearing: distinct from model default to surface ambient-env leakage in tests | Y |

**Risky keys found:** 0. Every key is either (a) load-bearing because Settings has no model default and would raise at import time, or (b) deliberately pinned to document a contract.

The pre-WP09 reasoning that found "load-bearing match" cases (e.g. `ENVIRONMENT`) is captured in v2.10-WP05's diagnosis — the conftest pins `ENVIRONMENT=development` so that the production fail-fast (WP05 A8) doesn't reject `DEV_AUTH_BYPASS=false` (the conftest also pins `DEV_AUTH_BYPASS=false`, matching the model default; both are required to keep the app importable in the test session AND keep WP05's fail-fast legal in production).

### Artifacts
- `tests/conftest.py` — every setdefault now carries `# load-bearing: <reason>`.
- `tests/test_conftest_env_audit_wp09.py` — AST scanner walks `tests/**/conftest.py`, flags any `os.environ.setdefault(...)` call missing a same-line `# load-bearing:` annotation. Self-tested with 3 synthetic cases.

3. **`fastapi.FastAPI(...)` (attribute form)** is intentionally not flagged today — no site exists in-tree. If one shows up, extend `_scan` in the C3 lint to match `Attribute(attr='FastAPI')` too.
