# v2.10-WP03 — Diagnosis of the "23 deferred auth/middleware tests" cluster

## TL;DR

**Classification: 100% bucket (a) — test-side fixture/mock-shape rot.** Every one
of the 23 failures was caused by tests that had not kept up with three
independent source-code refactors:

1. `app/auth/dependencies.py::require_owner_or_admin(resource_owner_id, user)`
   — signature is `(str, User)` but tests called it as `(user, owner_id)`.
   Production call sites (`app/routes/attachments.py`, `app/routes/problems.py`)
   already pass `(str, user)` correctly.
2. `app/auth/magic_link.py::send_magic_link(db, email, settings)` — `settings`
   is now an explicit positional arg. The module no longer imports
   `get_settings`. Tests still tried to `patch("app.auth.magic_link.get_settings", ...)`
   which raised `AttributeError: module ... does not have the attribute 'get_settings'`.
3. `app/middleware/rate_limit.py::magic_link_limiter` — the module-level
   singleton was renamed from `_limiter` to `magic_link_limiter`, and
   `check_magic_link_rate` is a sync function. Tests still patched
   `app.middleware.rate_limit._limiter` and awaited the dependency.

No production bug surfaced. Two minor request-mock and email-encoding bugs were
also fixed inline:

- `_make_request` mock set `headers["Authorization"]` (capital A) while
  `app/auth/dependencies.py` reads `request.headers.get("authorization", "")`
  (lowercase) — a plain-dict mock can't paper over Starlette's case-insensitive
  Headers. Lowercased the key in the test helper.
- `EmailMessage.set_content` quoted-printable encodes the body and wraps lines
  at ~76 chars with `=\n` continuations, which split the 43-char URL-safe
  token mid-string. Two tests that extracted the token from `str(message)`
  now read `message.get_content()` to get the decoded body.

## Per-ID root-cause classification (G1)

All 23 are bucket (a). Two underlying defects each cluster a chunk:

| Defect | Count | Test IDs |
| --- | --- | --- |
| `require_owner_or_admin` arg order | 4 | `TestRequireOwnerOrAdmin::test_owner_passes`, `::test_admin_passes_even_if_not_owner`, `::test_non_owner_non_admin_raises_403`, `::test_non_owner_non_admin_error_is_permission_related` |
| `_make_request` capital-A `Authorization` key | 1 | `TestGetCurrentUserTokenExtraction::test_falls_back_to_bearer_header_when_no_cookie` |
| `send_magic_link` needs explicit `settings` (no `get_settings` to patch) | 8 | `TestSendMagicLink::test_*` (all 8) |
| `verify_magic_link` `_mock_settings` patches a non-existent symbol | 8 | `TestVerifyMagicLink::test_*` (all 8) |
| Rate limiter singleton renamed `_limiter` → `magic_link_limiter`; `check_magic_link_rate` is sync | 2 | `TestCheckMagicLinkRateDependency::test_*` (both) |
| `EmailMessage` qp-wrapping splits token in body assertions | 2 (subset of send_magic_link 8) | `test_raw_token_never_stored_in_db`, `test_email_url_contains_base_url_and_token` |

**Bucket (b) source-code API drift:** technically all of the above are a flavour
of API drift, but because the *production* call sites were updated when the
APIs changed, the failures only live in tests. Classifying them as (a) per the
WP brief.

**Bucket (c) genuine production bug:** none.

## Per-file edit summary

### `tests/auth/test_dependencies.py`
- `_make_request`: `headers["Authorization"]` → `headers["authorization"]` (lowercase) with a comment pinning the production lookup as the source of truth.
- `TestRequireOwnerOrAdmin::test_*` (4 tests): swap `(user, owner_id)` → `(owner_id, user)` to match production signature.

### `tests/auth/test_magic_link.py`
- `_mock_settings()`: replaced the `patch("app.auth.magic_link.get_settings", ...)` (a now-undefined symbol) with a noop context manager. Settings are passed positionally instead.
- All 8 `send_magic_link` call sites: pass `_make_settings()` as the third argument.
- `test_raw_token_never_stored_in_db`: read `message.get_content()` instead of `str(message)` so quoted-printable `=\n` line continuations don't split the 43-char URL-safe token mid-string. Removed the silent `if match:` fallthrough — the assertion is now mandatory.
- `test_email_url_contains_base_url_and_token`: same `get_content()` fix.

### `tests/middleware/test_rate_limit.py`
- `patch("app.middleware.rate_limit._limiter", fresh)` → `patch("app.middleware.rate_limit.magic_link_limiter", fresh)` (replace_all, both occurrences).
- `await check_magic_link_rate(...)` → `check_magic_link_rate(...)` (sync function; `replace_all`).

### `tests/_v1_deferred.py`
- Removed the 23 IDs in scope. New count: 269.

## v2.11 follow-ups discovered

1. **Mock `Request` object is too lenient.** The fact that
   `request.headers["Authorization"]` silently failed for two years suggests
   tests should be using `starlette.datastructures.Headers` (case-insensitive)
   rather than plain dicts. Worth a v2.11 sweep of all `MagicMock`-based
   request fixtures.
2. **`require_owner_or_admin` argument order is footgun-shaped.** `(str, User)`
   is unusual — most FastAPI dependencies put the actor first. Consider a
   keyword-only signature in v2.11 to eliminate the class of bug that bit
   these four tests.
3. **`send_magic_link` settings-injection is fine, but `app/auth/jwt.py` and
   peers still use `get_settings()` internally** — inconsistent. Pick one
   pattern (injection-everywhere vs lookup-everywhere) and document it.
4. **EmailMessage qp-wrapping bites token extraction.** If we add more
   email-body assertions in v2.11, write a `decode_email_body(msg)` helper in
   `tests/helpers/` so future tests don't trip on the same `=\n` soft-break
   issue.

## Surprises

- The brief warned these were "mock-heavy unit tests" and "fixture drift". That
  was exactly right — no live-DB porting was needed, no migrations were
  involved, and the fixes were narrow rename/arg-swap operations.
- Two tests had a *silent* failure mode hidden behind `if match:` — they
  would have passed vacuously even if the assertion was conceptually broken,
  because qp wrapping made the regex fail to match. Fixed the assertions to
  be mandatory.
- The full-suite baseline landed on exactly the projected 918/274/14 counts —
  no surprise pass/skip count drift elsewhere.
