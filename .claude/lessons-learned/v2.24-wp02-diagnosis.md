Each commit can stand alone:

  and trivially reversible.
- WP02b changes ZERO runtime behaviour — the 1452-passing test suite
  was already validating that exact working-tree state before the commit

- WP02c–WP02d-docs are pure-doc commits with no Python/TS source impact.
- WP02d-alembic adds migrations whose effects are already present in

  suite has been green against those migrations for many WPs).
- WP02d-app + WP02d-tests track files whose presence on disk was already
  driving the 1452 P count. The commit just changes git's view, not
  Python's.
- WP02d-frontend tracks files that vite/vitest were already loading
  from disk for the 276 P count.

`git bisect` can now correctly attribute behaviour-affecting changes
to v2.24-WP03 onwards rather than colliding with prior-session leakage.
