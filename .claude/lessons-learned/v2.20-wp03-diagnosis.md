### B. `result =` reuse across heterogeneous `select()` calls

`services/solutions.py` reused the binding `result` for sequential
`await db.execute(...)` calls of different row shapes, locking mypy
into the first inferred row type and producing cascade
`attr-defined` errors against `Solution` for attributes that actually
live on `SolutionVersion`, `Problem`, or `User`. Renamed each binding
to a row-specific name (`sol_result`, `prob_result`, `actor_result`,
`version_result`, `max_result`, `unaccept_result`). No runtime change.

### C. Untyped `conds = []` in TicketService.list

`services/tickets.py:701` declared `conds = []` immediately before
appending `ColumnElement[bool]` values; mypy fixated on `BinaryExpression`
from the first append and rejected every subsequent `.append(...)`.
Annotated as `conds: list[Any] = []` — closed 8 `arg-type` errors.

### D. `sort_col` widening in `services/feed.py:_apply_sort`

After `Problem.activity_at` became `Mapped[datetime | None]`, the
implicit `sort_col` inference (originally `[datetime]` from the
first branch) no longer accommodated subsequent branches. Added
`sort_col: Any` declaration.

### E. Nullable `c.author_id` narrowing in `services/comments.py`

`Comment.author_id` is `Mapped[UUID | None]`. The dict lookup
`authors_by_id.get(c.author_id)` is now type-correct only with
explicit None-guard. Narrowed at the callsite.

## Genuine BY-DESIGN floor (51 residuals)

The remaining 51 entries split as:

* **`<Service>.list` self-shadow (7)** — `def list(self, ...)` on
  `TicketService` and `ProjectService` shadows the builtin `list`
  inside the class body, breaking subsequent
  `-> list[T]` annotations and (downstream) the route layer's
  iteration over the returned `list[T]`. Renaming the method to e.g.
  `list_tickets` is the correct future fix but touches 11+ external
  callers across MCP tools, routes, and tests — out of scope for a
  90-minute sweep.
* **`pg_insert(Model.__table__)` (7)** — `__table__` is a `FromClause`;
  the postgres dialect `insert()` overload set lists `Table` first.
  Either suppress with a cast or migrate to `pg_insert(Model)`. Real
  framework typing limit at the dialect surface.
* **`comments.py:167` return-value cluster (5)** — service builds a
  `dict[str, Any]` and the route's `-> CommentResponse` triggers
  `return-value`. Either build the schema instance at the service
  edge or relax the response_model. Out of scope here.
* **`solutions.py` attr-defined (3)** — three remaining lookups against
  joined-loaded relationships (`Solution.author.handle`,
  `Solution.upvotes[…]`); needs explicit `selectinload`-aware typing.
* **MCP / Starlette ASGI Mount signature (2)** — `app/mcp_server` and
  `app/main.py` mount points: Starlette's `Mount(app=...)` expects
  raw ASGI, but the project mounts a `Starlette` sub-app via
  FastAPI's `Request[State]` shape — genuine signature-variance limit
  noted in v2.19-WP02.
* **`Ticket.assignee_id` nullable / FK-pair coupling (2)** —
  `fanout_assigned()` arg types mark the assignee fields as required
  but they're co-nullable at the model. Fixable by either narrowing
  in `transition()` or relaxing the fanout signature — deferred.
* The remainder are scattered single-offender residuals — `due_soon_scanner`
  `Optional[datetime]` arithmetic, `routes/comments.py` schema-vs-dict
  return-value drift, etc.

These will not yield further allow-list reductions without disproportionate
refactors or signature changes that exceed a scoped sweep.

## Recommendation for v2.21

**More sweep available: YES — moderately.** Two clusters are still
mechanically fixable in a focused WP:

1. **`<Service>.list` rename** — ~11 caller updates across MCP tools,
   routes, and tests. Closes 7 valid-type + cascading downstream
   `routes/tickets.py:attr-defined` (7). Estimated 14-key reduction.
   *Unblocker*: an agreed rename target (`list_tickets` /
   `list_projects`) and willingness to touch caller test files.

2. **`pg_insert(Model.__table__)` → `pg_insert(Model)`** — 7 callsites,
   all in `services/ticket_notifications.py`. SA accepts ORM classes
   in dialect `insert()` overloads; just needs verification under
   tests. Estimated 7-key reduction.

If both land, the post-v2.21 floor would be ~30 keys — almost all
genuine framework limits (Starlette ASGI Mount variance, joined-load
attr-defined residuals).

If neither is feasible, the v2.20-WP03 floor of 51 keys is durable.
