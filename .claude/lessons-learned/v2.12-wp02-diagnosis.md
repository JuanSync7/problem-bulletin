# v2.12-WP02 (B1) — `tests/helpers/source_lint.py` extraction

## Framing

v2.11 closed seven separate source-shape lints (WP02, WP08-a, WP08-b,
WP09-a, WP09-b, WP10, WP11, WP12, WP15) — each one independently
rewrote the same plumbing: walk a tree of `.py` files, `ast.parse`
each, `ast.walk` for `Call`/`Constant(str)`/`JoinedStr` shapes, and in
WP08-a's case additionally reproduce `unittest.mock._get_target`'s
longest-importable-prefix walk. The duplication has reached critical
mass — the retrospective owes a helper. v2.12-WP02 ships
`tests/helpers/source_lint.py` (≤120 LOC) plus a port-in-place
migration of ≥6 existing consumers.

Pure refactor — no production code touched, no lint semantics changed
(if a latent bug surfaces in a migrated test, it's documented here and
deferred to a future WP). Pure-pytest only, no `pytest_collection_modifyitems`,
no conftest changes.

## End-goals

**G1. New `tests/helpers/test_source_lint.py` (TDD-red first) pins the
public API of `tests/helpers/source_lint.py`:**

- `iter_source_files(root, *, allow_list=None, suffix=".py")` — yields
  every `*.py` (or arbitrary suffix) under `root`, skipping
  `__pycache__` and `.venv`. `allow_list` is an iterable of paths
  relative to `root` to exclude.
- `parse_module(path)` — `ast.parse(path.read_text(), filename=str(path))`.
  Returns `None` if the file is unreadable or has SyntaxError (matches
  existing consumers' best-effort behaviour).
- `iter_string_literals(tree)` — yields every `ast.Constant` whose
  value is `str`, including those nested inside `ast.JoinedStr`.
- `iter_calls(tree, *, dotted_name=None)` — yields every `ast.Call`;
  if `dotted_name` is given, conservatively filters to calls whose
  `.func` resolves to that dotted name (matching the tail attribute
  against `ast.Name` or `ast.Attribute` chains).
- `resolve_patch_target(target)` — implements unittest.mock's
  `_get_target` algorithm: longest importable prefix, then walk
  remainder via `getattr`. Returns `(obj, last_attr)` on success,
  `None` on failure (does not raise).
- Self-tests for each helper using synthetic-bad inputs.

## Notes

- Pre-existing helper module `tests/helpers/__init__.py` already
  exists (v2.10-WP02). The new module sits next to `app_factory.py`,
  `requests.py`.
- `iter_source_files` accepts any iterable for `allow_list`; consumers
  pass either `list[str]`, `set[str]`, or `dict[str, str].keys()`
  (WP09 passes `dict.keys()`). The helper normalises via
  `set(allow_list or ())`.
- `resolve_patch_target` returns `None` for environmental import
  failures (`__SKIP__` sentinel in WP08-a is preserved as a separate
  `skip_reason` return). WP08-a's `_resolve_dotted` returns
  `(ok, detail)` — the helper exposes a richer 3-state return
  (`ok` / `unresolvable` / `skip`) to subsume the existing string-prefix
  hack cleanly.
