**No PIN has mechanical sweep targets.** Every catch-block, ts-any,
type:ignore, and typecheck entry carries a strong BY-DESIGN justification
that survives v2.18–v2.21 stale-detection sweeps. WP09 is explicitly
documented as a closed set.

ADDITIVE opt-in route list (29 routes today), and routes whose frontend
client currently uses `unknown` / inline shapes are deliberately omitted.

`export interface` / `export type` declarations not yet in `WP11_ROUTES`,

from the typical (c) "tighten a PIN" pattern. Refines forward rule (kk):
PIN value is the workflow; a workflow that's "fully BY-DESIGN" has no
sweep work — only coverage expansion.
