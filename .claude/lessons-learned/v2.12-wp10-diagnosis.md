## What changed

### Hook (`frontend/src/hooks/useSearchV2.ts`)

- Added two new return-shape fields: `refreshTotal: () => void` and `totalAuthority: "snapshot" | "live" | null`.
- Internally added a `refreshNonce` state and a `refreshPendingRef` ref. `refreshTotal()` flips the ref synchronously and bumps the nonce — the existing fetch effect picks up the nonce as a dependency and the next pass reads `refreshPendingRef.current` once (then clears it). This means:
  - The cursor stack is untouched (refresh is **not** navigation).
  - `refresh_total` only sticks for the one fetch the user asked for. Args-driven refetches (query change, cursor advance, filter toggle) do **not** inherit it.
- `totalAuthority` is derived from `activeArm.total_authority`, defaulting absent values to `"snapshot"` (matches the API client docstring). Returns `null` for `entity=all` — per-arm semantics don't combine cleanly across arms, so the All tab is opt-out by design (A5 covers `entity=all` if/when needed).

### Search page (`frontend/src/pages/Search.tsx`)

- Destructured `totalAuthority` and `refreshTotal` from `useSearchV2`.
- Banner rendered above the loader so the "Refreshing…" disabled state is visible during the refresh fetch (the loader replaces the result list, but the banner persists). Conditions: `activeTab !== "all"` && `hasPrev` && `totalAuthority === "snapshot"`. Uses `role="status"` + `aria-live="polite"`.
- One-shot semantics: if the server returns `snapshot` again, the banner stays; no auto-retry.

### CSS (`frontend/src/pages/Search.css`)

- Added `.search-snapshot-banner`, `.search-snapshot-banner__text`, `.search-snapshot-banner__btn` (+ hover/disabled). Left-edge `4px solid var(--color-info)` matches the existing info-pill idiom elsewhere in `App.css` (l.475).
