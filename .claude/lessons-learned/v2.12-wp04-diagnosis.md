## Per-file notes

### 1. `tests/e2e/test_otel_correlation_trace.py` — 1 latent bug (test-side)

Bug class: **test was depending on minimal-rig semantics that real `create_app()` overrides.**

Two issues surfaced when swapping to `build_test_app()`:

1. **Root-handler clobber.** Test wired its `io.StringIO` JSON log handler onto the root logger BEFORE calling `build_test_app()`. `create_app()` calls `setup_json_logging()` which does `root.handlers.clear() + addHandler(stdout)` — wiping the buf handler. Result: the assertion on `payload["trace_id"]` etc. failed because the in-span log never reached the buffer. **Fix:** build the app first, install buf handler second.
2. **SPA catch-all shadow.** Test registered `@app.get("/echo")` after `build_test_app()`. Production registers `@app.get("/{full_path:path}")` (SPA index) inside `create_app()`. FastAPI matches routes in registration order, so the SPA catch-all swallowed `/echo` and returned `index.html`. The handler under test never executed, so no "handling request" log was emitted. **Fix:** add the probe with `app.add_api_route(...)` then `app.routes.insert(0, probe)` so it precedes the catch-all. Renamed path to `/_t_echo` to make the test-only intent explicit.

Test now boots through the real `CorrelationIdMiddleware` from `create_app()` rather than its own ad-hoc `app.add_middleware(CorrelationIdMiddleware)` — net is closer to production. No production code touched.

## Risks observed (note for cluster 2-4)

1. **SPA catch-all shadowing.** Any test that adds its own route via `@app.get(...)` after `build_test_app()` will be intercepted by the SPA `/{full_path:path}` handler unless the route is explicitly inserted before it. Document this in the WP09 helper docstring if cluster 2-4 hit it again.
2. **`setup_json_logging` clears root handlers.** Tests that install custom log capture must do so AFTER `build_test_app()`.
3. **Lifespan note from the helper docstring (already documented):** without `with TestClient(app) as client:`, the lifespan never runs. None of the cluster 1 files needed background-task wiring, but cluster 2-4 may.
