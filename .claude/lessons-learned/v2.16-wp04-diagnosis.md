## Why N = 9 (above target N ≤ 6)

The remaining 9 entries fall into four genuine by-design buckets:

1. **Per-file/per-item aggregation loops** (Submit:200, ProblemDetail:931):
   the catch is *intentionally* silent because the outer caller aggregates
   per-iteration failure into a single user-facing list. Promoting these
   to `catch (err) { setError(err.message) }` would clobber the
   aggregated message on every iteration and lose the batch-surface
   contract.

2. **localStorage helpers** (Kanban/index.tsx:39, :46): cross-cutting
   helpers with no UI context. localStorage failures are environment-
   level (private mode, disabled storage, quota) and the helper's
   contract is "return null / fall back to defaults". Surfacing a toast
   from a helper that the Kanban page calls during render would be noise.

3. **Pure parse/format helpers** (Settings.tsx:40, :56): `new Date(iso)`
   throws on malformed input. Returning the raw ISO string is the idiomatic
   JS graceful-degradation pattern. No I/O, no envelope, nothing to thread.

4. **Best-effort optimistic / scanning sub-paths**
   (MentionsTab:246, ComponentDetail:53, Search:354): UI already
   communicates the failure mode (un-flipped row, "missing" state,
   empty filter dropdown), or the outer catch handles the user-visible
   error, or the surface is too noisy to surface every transient
   page-load 4xx into.

Each of these would degrade rather than improve UX if migrated. The target
(N ≤ 6) was a stretch goal; the honest count is 9.

## Allow-list comment style

All 9 surviving entries now start with `BY-DESIGN:` and include a one-line
rationale a future reader can grep / cite. The original WP02 comments
said things like "pre-WP03; UX TBD" which made every survivor look like
a deferred TODO; the new comments make clear these are intentional.
