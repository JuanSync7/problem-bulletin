# v2.10-WP02 — Diagnosis of the "21 deferred tests" cluster

## TL;DR

**Classification: (C) Mixed test rot — two distinct test-helper bugs, NOT a
production schema-drift bug.** The DB constraint `agent_accounts.created_by
NOT NULL` (introduced by migration `a17`) is correctly satisfied by every
production insert path. The 21 failing tests fall into two groups:

| Group | Count | Failure mode | Fix |
| --- | --- | --- | --- |
| (1) Agent-accounts seed rot | 15 | `NotNullViolationError on agent_accounts.created_by` from raw `INSERT INTO agent_accounts` helpers + `AgentAccountService.create_account` callers that pass no `created_by` | Test-helper change: seed helpers / callsites must set `created_by` |
| (2) Audit-log route-test rot | 6 | Two unrelated bugs in `tests/routes/test_audit_log.py`: a malformed JSON-bind SQL (`:meta::jsonb` is parameter-substituted then double-colon clashes) and a test FastAPI app built without app-level exception handlers | Test-helper change: fix bind syntax + wire exception handlers |

The WP01 manifest grouped all 6 audit_log tests under "Bucket D —
agent_accounts.created_by fixture rot". That classification is **wrong**:
none of the 6 audit_log tests insert into `agent_accounts`. The real cause is
documented above. We are still fixing all 21 in this WP because the audit_log
failures are similarly low-blast-radius test-helper bugs that ride along
cleanly.

## Evidence

### Migration history (the constraint)

`alembic/versions/a17_agent_accounts_created_by_not_null.py` (v2.5-WP34,
2026-05-19): conditional `ALTER COLUMN created_by SET NOT NULL`. Predecessor
`a16` is a backfill of NULL values to the oldest admin user. The column is
NOT NULL **in DB**.

### Model declaration (drift point)

`app/models/agent_account.py:35-37`:
```python
created_by: Mapped[UUID | None] = mapped_column(
    PgUUID(as_uuid=True), ForeignKey("users.id"), nullable=True,
)
```
The ORM model says `nullable=True`. This is application-vs-DB drift. The
SQLAlchemy insert succeeds at the ORM layer (no client-side check) and the
NotNull is enforced only at the asyncpg/DB layer.

### Production insert paths (all set `created_by`)

- `app/services/agent_accounts.py:42-74` — `AgentAccountService.create_account`
  accepts `created_by: UUID | None = None`. Defaults to None.
- `app/routes/admin/agent_accounts.py:55-60` — sole production caller; passes
  `created_by=actor.id` explicitly. **Production is correct.**
- No other production INSERT paths into `agent_accounts` exist (verified with
  `grep -rn "agent_account\|AgentAccount" app/`).

### Failing-test trace (group 1)

Repeatable for 15 tests, e.g. running
`tests/middleware/test_bearer_auth.py::test_bearer_agent_api_key_resolves_to_agent_actor`
with the deferral lifted:
```
sqlalchemy.exc.IntegrityError: ... NotNullViolationError: null value in
column "created_by" of relation "agent_accounts" violates not-null
constraint
DETAIL: Failing row contains (..., bot-aa2f11ea, null, ..., null, ...)
[SQL: INSERT INTO agent_accounts (id, name, handle, description,
api_key_hash, api_key_prefix, scopes, created_by, ...) VALUES (..., $8::UUID,
...)]
[parameters: (..., None, ...)]
```

### Failing-test traces (group 2 — audit_log)

`test_non_admin_gets_403`:
```
app/services/_admin.py:34: in require_admin
    raise PermissionDeniedError("Admin only")
E   app.services.exceptions.PermissionDeniedError: Admin only
```
The route raises correctly but the test app
(`tests/routes/test_audit_log.py::_build_app`) is built with bare
`FastAPI()`; it never registers the exception handlers from `app.main`.

`test_admin_gets_200_with_items_sorted_desc`:
```
sqlalchemy.exc.ProgrammingError: ... PostgresSyntaxError: syntax error at or near ":"
[SQL: ... VALUES ($1, $2, $3, $4, $5, :meta::jsonb, $6)]
```
`text()` substitutes `:meta` but leaves `::jsonb` as a stray colon-cast that
asyncpg can't parse. Needs `CAST(:meta AS jsonb)` or `:meta` typed via bindparam.

## Fix strategy

1. **Group 1 (15 tests):** introduce a single shared
   `tests/helpers/seed_agent_account.py` with `seed_agent_account(db, *,
   name, created_by=None, ...)` that auto-creates a seed user if
   `created_by` is None. Replace each `_mk_agent` / raw `INSERT INTO
   agent_accounts` helper to call it. Also pass `created_by=` to
   `AgentAccountService.create_account` in the service tests that go through
   the service.
2. **Group 2 (6 tests):** fix `_mk_audit_row` to use bindparam-typed JSONB
   (`bindparam("meta", type_=JSONB)`) and `_build_app` to register the
   exception handlers from `app.main.configure_exception_handlers` (or
   inline the two needed mappings).

## What we are NOT changing

- `alembic/versions/a17_agent_accounts_created_by_not_null.py` — the
  constraint reflects real domain intent (every agent must be auditably
  created by someone). Production is consistent with it.
- `app/services/agent_accounts.py:create_account(..., created_by=None)` —
  signature is internally consistent with the nullable ORM mapping and
  callers (admin route) already pass real values. Tightening the default to
  required would force a v2.11 API audit; deferred.
- `app/models/agent_account.py` — model drift (`nullable=True` vs DB NOT
  NULL) is annoying but harmless given (a) the ORM has no client check, (b)
  production callers all pass `created_by`. Logged as a follow-up but not
  blocking WP02.
