# v2.25-WP03 diagnosis

## WP03a: ProblemDetail.tsx:1052 onClick handler signature fix

### Bug
```
src/pages/ProblemDetail.tsx(1052,57): error TS2322:
  Type '(showLoading?: boolean) => Promise<void>' is not assignable to
  type 'MouseEventHandler<HTMLButtonElement>'.
  Types of parameters 'showLoading' and 'event' are incompatible.
    Type 'MouseEvent<HTMLButtonElement, MouseEvent>' is not assignable
    to type 'boolean | undefined'.
```

`fetchProblem` is defined (line 572) as:
```ts
const fetchProblem = useCallback(async (showLoading = false) => { ... }, [...]);
```

Line 1052 passed it bare to onClick:
```tsx
<button className="problem-detail__retry-btn" onClick={fetchProblem}>
```

React would pass a `MouseEvent` into the `showLoading` boolean slot —
truthy under JS semantics so it would have flipped the full-screen
spinner on instead of doing the inline retry. The TS bug masked a real
runtime UX bug.

### Fix
Single-line wrap in an arrow function so the event is dropped and
`showLoading` defaults to `false`:

```tsx
<button className="problem-detail__retry-btn" onClick={() => fetchProblem()}>
```
