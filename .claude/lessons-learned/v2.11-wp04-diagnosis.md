# v2.11-WP04 — Service-vs-route auth split + set_watch refresh + tag sort

## Scope

Three contract decisions surfaced during v2.10/v2.11 test-pinning:

- **A5** — `delete_attachment` had no service-layer auth; the route enforced
  uploader-or-admin via `require_owner_or_admin`. v1 tests pinned a
  service-layer 403/404 contract that production never had.
- **A6** — `set_watch` returns the SQLAlchemy identity-mapped row from the
  first call; on a second `set_watch(...)` for the same `(user, problem)` the
  `.level` field is stale unless the caller manually `db.refresh()`es.
- **A7** — `get_tags(sort=...)` silently fell back to name-order on any
  unknown value. The route (`GET /api/tags`) still returned 422, but direct
  service callers (MCP, scripts) had no way to detect the bad input.

## Decisions

| Bucket | Decision | One-line justification |
|---|---|---|
| **A5** | **(a) service is auth-aware** | One canonical guard at the boundary that all entry-points share (HTTP route, future MCP tool, bulk-delete job); eliminates the dead route-layer check and matches the only other auth-aware service (`tickets.py` already raises `ForbiddenError`). |
| **A6** | **refresh inside the service** | The bug is intrinsic to the upsert + identity-map interaction; pushing the fix to every caller is a footgun, and `db.refresh(watch)` after `db.flush()` is cheap (single PK lookup, same tx). |
| **A7** | **strict — raise `ValidationError`** | Consistent with the rest of the API surface (sprints, projects, components all raise on invalid input); the route already returns 422 so HTTP behaviour is unchanged; protects non-HTTP callers without surprising them. |

## What changed

### A5 — `app/services/attachments.py::delete_attachment`

- Added uploader-or-admin guard. Loads the actor `User` only when
  `actor_id != attachment.uploader_id` (single extra query, only on the
  cross-actor path).
- Raises domain `NotFoundError` (was `ValueError`) and `ForbiddenError`.
- Registered `NotFoundError -> 404` and `ForbiddenError -> 403` in
  `app.main._EXCEPTION_STATUS_MAP`. `ValidationError` deliberately NOT
  mapped because the tickets module owns a dedicated handler that emits a
  400 envelope; the public tags route still intercepts invalid sorts at
  the route layer.

### A5 — `app/routes/attachments.py`

- Removed the duplicate `require_owner_or_admin` call and the
  pre-existence `get_attachment` lookup. The route now delegates to the
  service and lets the global handler map exceptions.
- Removed the `ValueError -> HTTPException(404)` translation (the service
  raises `NotFoundError` directly).
- Dropped the now-unused `require_owner_or_admin` import.

### A6 — `app/services/watches.py::set_watch`

- Added `await db.refresh(watch)` after `db.flush()`. Existing callers
  that already refreshed (e.g. `test_set_watch_updates_existing_row`) are
  unaffected — refreshing twice is a no-op.

### A7 — `app/services/tags.py::get_tags`

- New module-level constant `_ALLOWED_TAG_SORTS = ("name", "usage")`.
- Raises `ValidationError([{"name": "sort", "reason": ...}])` on unknown
  sort. The matching field shape is the standard one used by
  `projects.py`/`sprints.py`/`components.py`.

## Tests

- **G1** `tests/services/test_attachments.py::test_delete_attachment_non_uploader_non_admin_raises_forbidden`
  — non-uploader / non-admin actor -> `ForbiddenError`, DB row intact,
  disk-removal helper never called.
- **G2** `tests/services/test_attachments.py::test_delete_attachment_not_found_raises_404`
  — rewritten from `pytest.raises(ValueError)` to
  `pytest.raises(NotFoundError)`.
- **G3** (existing) `test_delete_attachment_admin_can_delete_any` —
  retained, asserts admin path still succeeds; no route-level tests
  needed beyond the existing surface because the route is now a
  pass-through.
- **G4** `tests/services/test_notifications.py::test_set_watch_second_call_returns_fresh_level_without_manual_refresh`
  — asserts `second.level == WatchLevel.all_activity.value` *without* the

- **G5** `tests/services/test_admin.py::TestGetTags::test_raises_422_for_invalid_sort_param`
  — rewritten from "silent fallback to name order" to
  `pytest.raises(ValidationError)`.

## v2.11 follow-ups

- Audit other write-path services (`solutions.py`, `comments.py`,
  `voting.py`, `watches.py`, `notifications.py`) for the same "route does
  auth, service trusts the caller" pattern and decide where else to lift
  the check. Likely candidates: comment delete, solution delete.
- Consider whether `_EXCEPTION_STATUS_MAP` should also map
  `ValidationError -> 422` for non-ticket surfaces; today the ticket
  module's dedicated handler wins, so a service raising `ValidationError`
  from e.g. an admin/tags surface would 400 instead of 422 if it ever
  escaped the route layer. Currently moot because the route layer pre-
  validates, but worth a docstring.
- MCP tool surface for tags currently does not call `get_tags`; if/when
  it does, the strict A7 guard is exactly what we want — confirm in the
  MCP error envelope tests.
