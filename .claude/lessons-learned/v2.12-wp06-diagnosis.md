## Heuristics observed

- **WP04 heuristic #3 (drop redundant `EXCEPTION_HANDLERS` loop) was again
  the dominant pattern.** Files 6/7/8 all open-coded the same loop —
  this cluster matches WP05's experience exactly.
- **No SPA catch-all shadowing.** None of the eight files added their own
  test-only `@app.get(...)` after `build_test_app()`.
- **No log-handler ordering issues.** None of the cluster-3 files install
  a custom log buffer.
- **No assertion shape changes required.** The two files that asserted on
  error envelopes (`test_search_v2_cursors.py` checks `["detail"]`,
  `test_ticket_activity_cursor.py` checks `["error"]["code"]`) already
  matched what the real wiring emits — `HTTPException(400)` keeps the
  `{"detail": ...}` shape because it doesn't flow through `AppError`,
  while domain exceptions go through `AppError` and emit `{"error": {...}}`.
