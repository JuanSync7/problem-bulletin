# v2.10-WP09 — v1 `/api/search` cursor parity (DEPRECATE-FLAGGED)

**Status:** Investigation complete. **Recommended outcome: DEPRECATE v1
`/api/search` rather than port cursor pagination forward.** No code
changes landed — surfacing the call-site evidence for user review.

## TL;DR

The v1 `GET /api/search` endpoint has **zero live callers** in either
the frontend source tree or the backend codebase outside its own route.
Adding HMAC-signed cursor parity would consume engineering budget on a
dead surface. The task brief explicitly says: *"If v1 has zero frontend
callers and zero internal callers, STOP and report — recommend
deprecation, don't blindly port."* That clause fires here.

## G1 — Locate v1

- **Route**: `app/routes/search.py:170-191`, decorated `@router.get("")`
  on `APIRouter(prefix="/search")` mounted at `prefix=API` in
  `app/main.py:211` (`API` is `/api`). Resolved path: `GET /api/search`.
- **Service**: `app/services/search.py::search_problems` (full-text
  problems+solutions+comments rollup, REQ-350..360).
- **Contract**:
  - Params: `q`, `sort` (relevance|upvotes|newest), `category_id`,
    `tag_ids[]`, `status`, `limit` (1–100, default 20), `offset` (≥0).
  - Response: `{"results": [{problem_id, title, excerpt, rank,
    match_source, upstar_count, created_at}, ...]}` or
    `{"results": [], "message": "No results found"}`.
  - **No `total`. No `next_cursor`.** Pure offset pagination.

## G2 — Compare to v2

- **Route**: `app/routes/search.py:77-167` `GET /api/search/v2`.
- **Cursor model**: per-arm HMAC-signed opaque cursors via
  `app/services/_pagination.py::encode_signed_cursor /
  decode_signed_cursor`. Arm binding (`a` field) prevents replay across
  entity arms. Same JWT-layer secret.
- **Shape**: `{problems: {items, total, next_cursor}, tickets: {...},
  ...}` — five arms, plus shorthand `cursor=` for single-arm mode.
- **Parity question**: v1 returns a single de-duplicated stream
  (problems ∪ solutions ∪ comments rolled up to problems). Cursor
  parity would require either:
  1. Treating the stream as one synthetic "problems" arm (signed
     cursor over `(rank, problem_id)` since sort modes vary).
  2. Switching default sort to a stable keyset-friendly column. The
     current `rank DESC` sort is **not keyset-friendly** without a
     deterministic tie-breaker; `upvotes DESC` even less so.
- Either option is a non-trivial design lift, not a 50-line port.

### Backend
- `app/routes/search.py:19` imports `search_problems` and uses it only

  `/api/search` (grepped `app/`).

### Tests
- `tests/services/test_search.py` — exercises `search_problems`

  valuable even if the route is removed.

**Verdict**: zero live callers of `GET /api/search`. The endpoint
exists, is registered, but nothing in the running app or its
deployable frontend bundle (from current source) talks to it.

## G4–G6 — Implementation / Tests / Back-compat

**Skipped — gated on G3 decision.** Per the task brief, deprecation
recommendation halts the port. If the user wants to proceed regardless,
the design notes for porting are captured in G2 above; a sound
implementation would:
- Force a deterministic ORDER BY (e.g. `rank DESC, problem_id ASC`).
- Encode cursor payload `{r: <rank>, i: <problem_id>}` under arm
  `"problems_v1"` via `encode_signed_cursor`.
- Keep `offset` working alongside; reject combinations.
- Tests in `tests/routes/test_search_v1_cursor.py`: HMAC roundtrip,
  expired/tampered cursor → 400, query-change implicit reset (caller
  responsibility, just verify cursor-arm-mismatch is rejected),
  end-of-results returns `next_cursor=null`.

## Recommendation (for user)

**Deprecate `GET /api/search`.** Concrete plan:

1. **Deprecation header** in `app/routes/search.py` v1 handler:
   `response.headers["Deprecation"] = "true"` and
   `response.headers["Link"] = '</api/search/v2>; rel="successor-version"'`.
2. **Mark response** with `Sunset: <date 90d out>` per RFC 8594.
3. **Add structured log** when the endpoint is hit so we can
   confirm the zero-caller assertion in prod telemetry over a
   monitoring window.
4. **After monitoring window with zero hits**, remove the route, the
   `search_problems` service (or downgrade to a private helper if its
   tests retain value), the `mock/api.ts` entry, and the stale
   `frontend/dist` artifact.
5. **Track as v2.11-WP-XX** "Sunset v1 /api/search".

If the user disagrees and wants parity ported, reopen WP09 with that
explicit instruction; G2 has the design sketch.
