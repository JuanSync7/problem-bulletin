# v2.18-WP04 Diagnosis: L4 `# type: ignore[arg-type]` in notifications_v1.py

## Site
`app/routes/notifications_v1.py:177` — orphan-actor synthesis branch in
`GET /api/v1/notifications` (`list_notifications`). When `_hydrate_actors`
returns no `PersonRef` for `(row.actor_type, row.actor_id)` (deleted
user/agent), the route constructs a stand-in `PersonRef(kind=..., ...)`.

## Row source
`row` is a `TicketNotification` ORM model instance returned by
`ticket_notifications_service.list_for_recipient(...)` (which delegates to
`list_for_actor` for agent recipients). Both yield `list[TicketNotification]`
inside the `items` of the service result dict.

`TicketNotification.actor_type` is declared in
`app/models/ticket_notification.py:59` as:

```python
actor_type: Mapped[str] = mapped_column(Text, nullable=False)
```

It is *not* a SQLAlchemy Enum column — it is plain `Text` with a check
constraint (`actor_type IN ('user','agent')`, line 40) enforcing the domain
at the DB level. So mypy sees `str`, while
`PersonRef.kind: Literal["user", "agent"]` (`app/schemas/people.py:34`)
demands a narrower type.

## Why the ignore originally
The previous LEGACY justification described it as "Optional[str] from the
SQL row". In reality the column is `Mapped[str]` (non-nullable, not
Optional), but the type-checker mismatch is still real: `str` is wider
than `Literal["user", "agent"]`.

## Approach chosen: (a) `typing.cast()`
This is the **only callsite** in `notifications_v1.py` that needs to
narrow `row.actor_type` to the PersonRef enum slot — the surrounding code
just uses `row.actor_type` as a string (tuple key on line 173). Tightening
the model column type (option b) would require either:
- migrating `actor_type` to `Mapped[ActorType]` everywhere (an `Enum`-backed
  column rewrite touching the model, migrations, and every consumer), or
- adding a TypedDict / dataclass at the service boundary just for this
  one read path.

Both are disproportionate to one orphan-fallback line. The DB check
constraint already guarantees the runtime invariant — `cast()` documents
the assertion without changing runtime behaviour.

## Diff
- Added `cast` to the `from typing import ...` line.
- Replaced `kind=row.actor_type,  # type: ignore[arg-type]` with
  `kind=cast(Literal["user", "agent"], row.actor_type),`.
- Removed the `app/routes/notifications_v1.py:177` entry from
  `_OFFENDER_ALLOWLIST` in `tests/test_type_ignore_lint_wp03_v217.py`
  (the LEGACY: type: ignore section is now empty).
