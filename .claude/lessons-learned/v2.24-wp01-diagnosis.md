## TicketAttachmentRead recon (Option c)

### Schema (`app/schemas/tickets.py:218-230`)

```python
class TicketAttachmentRead(BaseModel):
    model_config = ConfigDict(from_attributes=True, extra="allow")

    id: UUID
    ticket_id: UUID
    uploaded_by: UUID
    uploaded_by_type: Literal["user", "agent"]
    filename: str
    content_type: str
    byte_size: int
    storage_path: str
    agent_step_id: str | None = None
    created_at: datetime
```

**Field count: 10** (matches v2.23 seed). All fields required at OpenAPI level except `agent_step_id` (nullable, defaults to `None`). `extra="allow"` mirrors the watcher schema pattern.

### Route + envelope

* **List route**: `GET /api/v1/tickets/{id_or_key}/attachments`
* **Location**: `app/routes/tickets.py:786`
* **Decorator**: `@router.get("/{id_or_key}/attachments", response_model=Page[TicketAttachmentRead])`
* **Envelope**: `Page[TicketAttachmentRead]` — emits OpenAPI schema `Page_TicketAttachmentRead_` (wrapper has same `{items, next_cursor, total}` shape as `Page_TicketWatcherRead_`)
* Also adjacent: `POST /{id_or_key}/attachments` (multipart upload, line 803) and `DELETE /{id_or_key}/attachments/{attachment_id}` (line 831) — not in scope for the read-side parity pin.

### Frontend state

* `frontend/src/api/attachments.ts` does **not** exist in the working tree.
* `frontend/src/api/tickets.ts` (last commit 27c0d3b, v2.23-WP03) contains **no** `TicketAttachment` interface, no `attachments` helper, no `any`-typed parse of attachments.
