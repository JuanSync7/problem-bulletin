# v2.10-WP13 — TipTap extension audit (diagnosis)

## Goal
WP64 (v2.9) fixed one StarterKit duplicate-extension warning (`link` +
`underline` in v3 silently bundled, also imported individually).
Sweep the codebase to confirm zero other extension duplicates and zero
hidden console warnings from TipTap editor instantiation.

## G1 — Inventory of TipTap consumers (frontend/src)

Search criteria: `useEditor`, `EditorContent`, `from '@tiptap`.

Result: **exactly one consumer** — `frontend/src/components/RichEditor.tsx`.
Everything else is either a test (`RichEditor.test.tsx`) or unrelated.

### `frontend/src/components/RichEditor.tsx`

Imports:
| Import | Source |
| --- | --- |
| `useEditor`, `EditorContent`, `Editor` (type) | `@tiptap/react` |
| `StarterKit` | `@tiptap/starter-kit` |
| `Underline` | `@tiptap/extension-underline` |
| `Link` | `@tiptap/extension-link` |
| `Markdown` | `tiptap-markdown` |

StarterKit configuration (line ~194):

```ts
StarterKit.configure({ link: false, underline: false }),
Underline,
Link.configure({ openOnClick: false, autolink: true }),
Markdown.configure({ ... }),
```

Both StarterKit-bundled v3 duplicates (Link, Underline) are **already
disabled in the StarterKit config** so the explicitly-imported variants
can register cleanly. This is the WP64 fix.

## G2 — Cross-check imported extensions against StarterKit v3 bundle

StarterKit v3 bundles (per https://tiptap.dev/docs/editor/extensions/functionality/starterkit):
Bold, Code, CodeBlock, Document, Dropcursor, Gapcursor, HardBreak,
Heading, History, HorizontalRule, Italic, ListItem, OrderedList,
BulletList, Paragraph, Strike, Text, **Link**, **Underline**.

Cross-check against RichEditor's individual imports:

| Individually imported | In StarterKit v3 bundle? | StarterKit disabled? | Verdict |
| --- | --- | --- | --- |
| `Underline` (`@tiptap/extension-underline`) | yes | yes (`underline: false`) | OK — no duplicate |
| `Link` (`@tiptap/extension-link`) | yes | yes (`link: false`) | OK — no duplicate |
| `Markdown` (`tiptap-markdown`) | no | n/a | OK — third-party, not in bundle |

No further individual `@tiptap/extension-*` imports exist anywhere in
the frontend (verified via grep `from '@tiptap`). All other extension
names (Bold, Italic, Strike, Code, CodeBlock, Heading, BulletList,
OrderedList, Blockquote, HorizontalRule, etc.) are referenced **only
through the `editor.chain().focus().toggleX()` API** — i.e. used at
runtime via the StarterKit-bundled versions. No second registration.

**Duplicates beyond WP64: none.**

## G3 — Fixes required

None. The WP64 fix is sufficient and complete.

## G4 — Console-warning sentinel test

Plan: extend `RichEditor.test.tsx` with a sentinel that spies on
`console.warn` and `console.error` across the editor's mount lifecycle
(synchronous mount + post-onCreate microtask) and asserts neither was
called with anything originating from TipTap. This is the canary that
will RED if a future WP re-introduces a duplicate extension.

Implementation note: TipTap emits "Duplicate extension names found"
via `console.warn`. We assert `console.warn` is not called at all
during editor mount. React Router future-flag warnings observed in
the baseline test suite come from unrelated tests (LabelDetail's
MemoryRouter); the RichEditor test does not use a Router so its
console namespace is clean.

- `@tiptap/pm`: `^3.23.4`

- `@tiptap/starter-kit`: `^3.23.4`
- `tiptap-markdown`: `^0.9.0`

**Major version: TipTap v3.**

Bundling change vs v2 (relevant for future-proofing):
- v2 StarterKit did **not** bundle `Link` or `Underline`. Code that
  worked on v2 silently broke on v3 → WP64.
- If we ever upgrade to a future v4, re-audit the StarterKit bundle
  list. The safe pattern going forward: every individually-imported
  extension that overlaps with StarterKit should be explicitly
  disabled in `StarterKit.configure({...})`, even if v3 docs don't
  require it (defence in depth).
