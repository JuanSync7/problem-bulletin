## 3. Pydantic-plugin-eliminated keys (25)

Grouped by original rationale cluster:

### Pydantic-cluster (5)
Errors the plugin directly fixes (BaseSettings constructor, Page[T] generic
item-type variance):
- `app/config.py:172:call-arg`
  - Original: *Pydantic v2 BaseSettings populates fields from env vars; mypy
    cannot model the zero-arg constructor without the pydantic.mypy plugin
    (currently broken against mypy 1.20.1).* — plugin now models it.
- `app/routes/agents.py:108:arg-type`
- `app/routes/projects.py:88:arg-type`
- `app/routes/projects.py:235:arg-type`
- `app/routes/sprints.py:53:arg-type`
  - Original: *Pydantic v2 Page[T] item-type drift -- routes pass
    `[t.to_dict() for t in rows]` (list[dict]) to Page[XRead]; runtime
    validation coerces.* — plugin narrows Page[T] item-type and the
    `list[dict] → Page[XRead]` flow now types correctly.

### SQLAlchemy-cluster downstream unblocks (20)
The plugin doesn't touch SQLAlchemy directly. These errors disappeared
because the Pydantic plugin gives mypy enough type info on the
schema-side of route handlers that the chained-argument inference no
longer fails at the SQLAlchemy boundary. Once the schema arg types
narrow, the descriptor-resolution path used to compute the call-site
type evidently resolves without falling back to `Column[T] | None`.
- `app/routes/auth.py:98,99,101,117,118,120:arg-type` (6)
- `app/routes/notifications.py:96,100:arg-type` (2)
- `app/routes/notifications_v1.py:84,85,86,87:arg-type` (4)
- `app/routes/solutions.py:201,202,203,205:arg-type` (4)
- `app/services/audit_log.py:185,186,187,188:arg-type` (4)
  - All originally classified: *BY-DESIGN: SQLAlchemy Column[T] / Mapped[T]
    leakage past the ORM boundary -- mypy cannot resolve the descriptor
    return type without the SQLAlchemy plugin; `arg-type` is the resulting
    artefact.* The rationale was partially wrong: the surrounding Pydantic
    inference, not the SQLAlchemy descriptor, was the proximate blocker.

This confirms WP01's hypothesis that the plugin would clear more than the 6
nominal "Pydantic-cluster" keys.
